# AWS_Concept — AWS 핵심 서비스

```txt
AWS = Amazon Web Services
클라우드 컴퓨팅 서비스 플랫폼
필요한 자원을 온디맨드로 사용 → 인프라 직접 구축 없이 서비스 운영 가능
```

---

# EC2 — Elastic Compute Cloud ⭐️

```txt
AWS 에서 제공하는 클라우드 기반 가상 서버
컴퓨팅 리소스를 탄력적으로 제공하는 서비스
```

```txt
특징:
  다양한 인스턴스 유형
    일반 목적 / 컴퓨팅 최적화 / 메모리 최적화 등 용도에 맞게 선택
  확장성
    수요에 따라 인스턴스 수를 늘리거나 줄임 → 비용 효율적 운영
  제어 및 관리
    운영 체제 / 네트워크 설정 / 보안 등 서버 환경을 세부적으로 구성 가능
```

# RDS — Relational Database Service ⭐️

```txt
클라우드에서 관계형 데이터베이스를 손쉽게 설정·운영·확장할 수 있는 관리형 서비스
DB 서버를 직접 구축하지 않아도 됨
```

```txt
지원 엔진:
  Amazon Aurora / PostgreSQL / MySQL / MariaDB / Oracle / SQL Server

특징:
  자동화된 관리   백업 / 패치 / 복구 자동 처리
  고가용성        멀티 AZ 배포 → 장애 발생 시 자동 전환
  읽기 복제본     읽기 성능 향상
  보안            VPC 내 격리 / IAM 액세스 제어 / 데이터 암호화
```

# VPC — Virtual Private Cloud ⭐️

```txt
AWS 클라우드 내에 사용자가 정의한 가상 네트워크
리소스를 격리된 네트워크 안에 배포 가능
```

```txt
특징:
  네트워크 구성 제어
    IP 주소 범위 / 서브넷 / 라우팅 테이블 직접 설정
  보안 제어
    보안 그룹 / 네트워크 ACL 로 인바운드·아웃바운드 트래픽 제어
  하이브리드 연결
    온프레미스 네트워크와 VPN / Direct Connect 로 연결 가능
```

# ELB — Elastic Load Balancer ⭐️

```txt
들어오는 트래픽을 여러 대상(EC2 인스턴스 등)으로 자동 분산
애플리케이션의 가용성과 확장성 향상
```

```txt
종류:
  Application Load Balancer (ALB)
    HTTP / HTTPS 트래픽 → 레이어 7 로드 밸런싱
    URL / 헤더 기반 라우팅 가능

  Network Load Balancer (NLB)
    고성능 TCP 트래픽 → 레이어 4 로드 밸런싱

  Gateway Load Balancer
    서드파티 가상 어플라이언스와 통합

특징:
  자동 확장   트래픽 변화에 따라 자동 확장·축소
  헬스 체크   비정상 인스턴스를 자동으로 제외
```

# Route 53 ⭐️

```txt
확장성과 가용성이 높은 클라우드 DNS 서비스
도메인 이름 → IP 주소 변환 + 트래픽 라우팅
```

```txt
특징:
  도메인 등록    새로운 도메인 이름 등록 및 관리
  DNS 라우팅
    지연 시간 기반 / 가중치 기반 / 지리 위치 기반 등 다양한 라우팅 정책
  헬스 체크
    엔드포인트 가용성 확인 → 비정상 시 트래픽을 대체 리소스로 자동 라우팅
```

# Elastic Beanstalk

```txt
애플리케이션 배포를 단순화하는 관리형 서비스
코드만 업로드하면 인프라 프로비저닝 / 로드 밸런싱 / 스케일링 자동 처리
```

```txt
특징:
  자동화된 관리   서버 설정 / 환경 구성 / 모니터링 자동 처리
  커스터마이징    필요에 따라 세부 설정 조정 가능
  배포 옵션       롤링 업데이트 / 블루·그린 배포 등 다양한 전략 지원
```

# Lightsail

```txt
소규모 프로젝트 / 간단한 웹 애플리케이션을 빠르게 시작하는 서비스
가상 서버 + 스토리지 + 데이터 전송을 하나의 패키지로 제공
```

```txt
특징:
  간단한 사용법   복잡한 설정 없이 몇 번의 클릭으로 리소스 생성
  예측 가능한 비용  월별 고정 요금제 → 비용 관리 용이
  사전 구성 템플릿  WordPress / LAMP / MEAN 등 인기 스택 제공

EC2 와 차이:
  EC2   세부 제어 가능 / 복잡 / 유연
  Lightsail  간단 / 고정 요금 / 소규모 프로젝트 적합
```

# IAM — Identity Access Management ⭐️

```txt
AWS 리소스에 대한 액세스를 안전하게 제어하는 서비스
누가 / 어떤 리소스에 / 어떤 행동을 할 수 있는지 관리
```

```txt
특징:
  사용자 및 그룹 관리   개별 사용자 생성 및 그룹화
  권한 제어             세부적인 권한 정책으로 리소스 접근 제어
  MFA                   다단계 인증으로 추가 보안 계층 설정

최소 권한 원칙:
  필요한 권한만 부여
  루트 계정은 일상적으로 사용하지 않음
```

# S3 — Simple Storage Service ⭐️

```txt
확장성이 뛰어난 객체 스토리지 서비스
대용량 데이터를 안전하고 저렴하게 저장·검색 가능
```

```txt
특징:
  높은 내구성    데이터를 여러 시설에 중복 저장 → 99.999999999% 가용성
  스토리지 클래스
    S3 Standard          자주 접근하는 데이터
    S3 Intelligent-Tiering  접근 패턴에 따라 자동 이동
    S3 Glacier           장기 보관 / 저렴
  보안           버킷 정책 / ACL / IAM 연동으로 세부 접근 권한 관리
  라이프사이클    저장 기간에 따른 자동 이동 또는 삭제 정책 설정
```

# AWS 서비스 한눈에

|서비스|역할|한 줄 요약|
|---|---|---|
|`EC2`|가상 서버|내 서버를 클라우드에서 빌려 쓰는 것|
|`RDS`|관리형 DB|DB 서버 직접 안 만들어도 됨|
|`VPC`|가상 네트워크|AWS 안의 내 전용 네트워크|
|`ELB`|로드 밸런서|트래픽을 여러 서버로 분산|
|`Route 53`|DNS|도메인 → IP 연결 + 트래픽 라우팅|
|`Elastic Beanstalk`|배포 자동화|코드만 올리면 나머지 자동 처리|
|`Lightsail`|간단한 서버|EC2 보다 쉽고 고정 요금|
|`IAM`|권한 관리|누가 뭘 할 수 있는지 제어|
|`S3`|객체 스토리지|파일 / 이미지 / 백업 저장|
