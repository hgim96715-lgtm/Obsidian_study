# 김한경  Next.js NestJS로 화면과 API 데이터 흐름을 함께 만드는 개발자

## 자기소개서

안녕하세요. 사람이 손으로 하던 일을 서비스로 옮기고, 데이터가 제대로 쌓이게 만드는 개발자 김한경입니다.

저는 화면에서 출발해 API와 데이터베이스까지 흐름을 이어가는 개발에 관심이 있습니다. 일상이지에서 퍼블리셔로 근무하며 교육 콘텐츠 스토리보드를 분석하고, HTML · CSS · JavaScript 기반 인터랙티브 미니게임을 구현했습니다. 학습자의 클릭·입력·선택에 따라 화면 상태가 바뀌는 기능을 만들었고, Array/JSON 기반 문제 데이터를 구성하며 사용자 정답 검증 로직을 직접 짰습니다. 화면을 만드는 일이었지만, 매번 마주한 질문은 **데이터를 어떻게 설계하고 처리할 것인가**였습니다. 기획서 요구사항을 화면 동작 단위로 쪼개며 데이터 구조와 UI가 맞물리는 흐름을 경험했고, 그 경험이 지금의 개발 공부로 이어졌습니다.

이후 NestJS와 PostgreSQL을 중심으로 백엔드 역량을 쌓았습니다. 영화·감독·장르 도메인 REST API 프로젝트에서 TypeORM 관계형 모델링, JWT 인증, RBAC, 트랜잭션, 파일 업로드, Jest 테스트, AWS 배포까지 다뤘고, 이후 Prisma로 점진 전환하며 ORM 선택과 스키마 설계를 직접 비교해 보았습니다. Artinerary에서는 문화누리 공공 API의 XML을 수집·정규화해 전시 카탈로그를 만들었고, Music Community에서는 pnpm 모노레포로 NestJS에 도메인·인증·집계를, Next.js에 UI를 두어 피드·가입/로그인·관리자 통계까지 구현한 뒤 Vercel · Railway · Neon에 배포했습니다. 요청이 API를 거쳐 DB에 저장되고 다시 화면으로 돌아오는 전 과정을 직접 맞춰 본 경험입니다.

SQLD를 취득하며 SQL과 관계형 데이터베이스 기초를 다졌고, 공공 API 기반 데이터 수집·적재 프로젝트(서울역 열차 운행, 응급실 병상 현황 역설계)를 통해 데이터가 수집에서 저장·활용까지 이어지는 흐름에도 관심을 넓혀 가고 있습니다. 공부한 내용은 Obsidian 위키에 동작 원리와 트러블슈팅 중심으로 정리하며, 프로젝트에 바로 반영하는 습관을 유지하고 있습니다.

기술적 어려움을 막막한 벽이 아니라 풀어야 할 단계로 봅니다. 문제를 작게 나누고, 원인을 추적하며 끝까지 해결하는 과정에서 성장해 왔습니다. 앞으로도 꾸준히 배우고 기록하며, 안정적인 API와 신뢰할 수 있는 데이터 흐름을 만드는 개발자로 성장하겠습니다.
